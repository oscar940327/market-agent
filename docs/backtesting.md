# Backtesting

這份文件說明 Market Agent 的策略回測在做什麼。

## 解釋

Backtesting 是用歷史價格資料檢查某個技術策略以前表現如何。

它不是預測未來，也不是保證策略有效。  
它的用途是提供「歷史證據」，讓研究報告不要只停留在當下技術指標。

> 問「MU 突破策略以前表現怎麼樣」。系統會用過去約 15 年的價格資料回測突破策略，整理交易次數、勝率、平均報酬、最大虧損和證據品質。這讓策略不是只看今天的訊號，而是能知道類似規則過去大概表現如何。

## 支援的策略

目前主要支援三種技術策略：

| 策略 | 問法範例 | 想觀察什麼 |
| --- | --- | --- |
| `breakout` | `MU 突破策略以前表現怎麼樣` | 價格突破前高後，短期表現是否有優勢。 |
| `volume_surge` | `MU 放量策略以前表現怎麼樣` | 成交量明顯放大時，後續是否容易延續。 |
| `pullback` | `MU 拉回策略以前表現怎麼樣` | 股價接近 MA20 等支撐區時，是否有反彈機會。 |

## 回測會輸出什麼

回測報告會整理：

- 資料範圍
- 總交易次數
- 勝率
- 平均報酬
- 最大虧損
- 證據品質
- 範例交易
- 風險提醒

這些資訊會幫助使用者判斷：這個策略過去是否有參考價值，以及風險是否過高。

## 證據品質怎麼看

Backtest evidence quality 不是只看勝率。

系統還會看：

- 樣本數夠不夠
- 是否有約 15 年歷史資料
- 是否跨過不同市場環境
- 平均報酬是否為正
- 最大虧損是否過大

例如：

- 樣本很多，但最大虧損很高，證據品質仍可能只給 `medium`。
- 上市不到 15 年的股票，即使策略看起來不錯，也會標記歷史資料不足。
- 沒有樣本時，不會硬產生結論。

## 為什麼 Backtest 不用 ML Reference

策略回測問題主要回答：

```text
這個策略以前表現怎麼樣？
```

它看的是歷史交易規則本身，不是模型今天對未來的 prediction。

所以 backtest workflow 會顯示：

```text
ML Reference: not used
```

這是正常狀態，不是資料缺失。

## 跟單股分析的差別

單股分析回答的是：

```text
這檔股票現在狀態如何？
```

策略回測回答的是：

```text
某個策略規則以前表現如何？
```

兩者可以互補。

例如單股分析發現目前沒有突破訊號，就不會附加突破歷史證據。  
但如果你直接問突破策略回測，系統仍可以用歷史資料回答這個策略過去的整體表現。

## 目前限制

目前回測仍是第一版，主要限制包括：

- 尚未完整納入交易成本、滑價、稅費。
- 尚未完整做 position sizing。
- 尚未完整做停損、停利與資金曲線。
- 目前重點是策略訊號的歷史參考，不是完整交易系統。
- 回測結果只代表過去，不代表未來會重複。

## Takeaways

- Backtesting 讓策略分析有歷史證據。
- 勝率不是唯一重點，最大虧損和樣本品質同樣重要。
- Backtest 不使用 ML Reference，因為它回答的是歷史策略表現。
- 目前回測是研究輔助，不是完整交易系統。
